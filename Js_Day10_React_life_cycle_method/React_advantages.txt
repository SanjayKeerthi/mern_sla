1. What is React, and what problem does it solve compared to traditional DOM manipulation?
React is a JavaScript library used to build fast, interactive user interfaces, especially for single-page applications.

Problem with traditional DOM manipulation:
Direct DOM updates are slow
Developers manually update UI in many places
Code becomes hard to maintain as apps grow

How React solves this:

Uses a component-based architecture
Uses a Virtual DOM to minimize real DOM updates
UI automatically updates when state changes

üëâ React makes UI development faster, cleaner, and more predictable.

2. What is JSX in React, and how is it different from regular HTML?
JSX (JavaScript XML) is a syntax extension that allows writing HTML-like code inside JavaScript.

const element = <h1>Hello, React!</h1>;

| Feature     | JSX                    | HTML              |
| ----------- | ---------------------- | ----------------- |
| Used in     | JavaScript             | Browser           |
| Attributes  | `className`, `htmlFor` | `class`, `for`    |
| Expressions | `{}` allowed           | Not allowed       |
| Compilation | Transpiled by Babel    | Directly rendered |

Example:-
<h1>{userName}</h1>

4. What is the Virtual DOM, and how does React use it to update the UI efficiently?
The Virtual DOM is a lightweight JavaScript copy of the real DOM.

How it works:
React creates a Virtual DOM tree
When state changes, a new Virtual DOM is created
React compares old vs new (diffing)
Only the changed elements are updated in the real DOM

üëâ This process is called reconciliation, and it makes React fast and efficient.
